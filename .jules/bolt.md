## 2024-05-18 - Replacing Enumerable LINQ methods with Spans and Range indexing
**Learning:** In C#12+ with Collection Expressions (`[.. a, .. b]`), replacing `[.. arr.Take(n)]` with `[.. arr.AsSpan(0, n)]` yields significant performance gains by avoiding the allocation of the enumerator and the LINQ interface dispatch overhead, while still staying within the guidelines of using declarative structures and avoiding manual loops/`Array.Copy`. However, per AGENTS.md, I MUST avoid replacing standard LINQ methods like `.Where()` with imperative manual loops or `Array.FindAll` or similar manual tracking.
**Action:** Use `.AsSpan(...)` instead of `.Take(...)` in hot paths within collection expressions for array manipulation to optimize without breaking AGENTS.md directives.
## 2024-06-16 - Array Contains vs Array.IndexOf Performance in Graph Operations
**Learning:** Replaced `.Contains()` with `Array.IndexOf(arr, val) != -1` in `MemoHandlR.cs` and `MemoBase.cs` during hot path graph operations (observer updates). In the context of `MemoizR` dependency graph updates, this avoids LINQ extensions enumerator allocation overhead and leverages faster array traversal methods, which is crucial for performance given the frequent evaluations of node states in read/write scenarios. This complements the `.AsSpan()` optimizations previously recorded without violating AGENTS.md constraints against imperative looping over LINQ.
**Action:** Use `Array.IndexOf(arr, val) != -1` instead of `.Contains(val)` for existence checks on array backing fields to reduce allocations in frequently evaluated paths.
