## 2024-05-15 - [Linked CancellationTokenSource Memory Leak]
**Vulnerability:** A linked `CancellationTokenSource` was created in `StructuredRaceJob` but never disposed. Because it was linked to a longer-lived parent `CancellationTokenSource` from the global `Context`, the parent maintained a strong reference to the linked source's callbacks, preventing garbage collection.
**Learning:** In .NET, `CancellationTokenSource.CreateLinkedTokenSource` attaches an internal callback to the parent token(s). Failing to dispose the linked source leaves this callback registered, leading to an unbounded memory leak if many such sources are created over the lifetime of the parent token. This creates a Denial of Service (DoS) vulnerability through resource exhaustion in long-running concurrent scenarios.
**Prevention:** Always ensure that `CancellationTokenSource` objects, especially those created via `CreateLinkedTokenSource`, are explicitly disposed. In the context of `MemoizR.StructuredConcurrency`, they must be passed to the current `IStructuredResourceGroup` via `AddResource()` so their lifecycle is correctly managed by the structured concurrency mechanism.
